# Distributed Trading Engine (Birzha)

Асинхронный распределенный торговый движок на Kotlin, построенный на принципах **Event Sourcing**, **CQRS** и **Actor Model**.

## 🚀 Особенности

- **Высокая производительность**: Использование Kotlin Coroutines и Channels для обработки тысяч ордеров в секунду.
- **Распределенность**: Кластер из нескольких нод с поддержкой Leader Election (имитация) и репликации событий.
- **Event Sourcing**: Полное восстановление состояния (Order Book, Balances) из истории событий.
- **CQRS**: Четкое разделение Command Side (прием ордеров) и Query Side (read-модели для UI/API).
- **Гарантии консистентности**:
    - **Double-Entry Accounting**: Балансы пользователей всегда под строгим контролем.
    - **Exactly-once processing**: Идемпотентность команд и дедупликация событий.
    - **Deterministic matching**: Результат матчинга не зависит от времени или случайности.
- **Отказоустойчивость**: Система переживает падение нод и восстанавливается через Replay логов.

## 🏗 Архитектура

### Компоненты
- **ClusterNode**: Узел кластера. Может быть лидером (принимает команды) или фолловером (реплицирует состояние).
- **MatchingActor**: Акторы (по одному на инструмент), работающие в изолированных корутинах. Исключают race conditions без блокировок.
- **OrderBook**: Чистая логика матчинга (Price-Time Priority).
- **BalanceService**: Управление деньгами (USD, BTC) с механизмом холдирования (Lock/Unlock) под ордера.
- **EventStore**: Append-only хранилище с оптимистичными блокировками.

## 🛠 Запуск и тестирование

### Требования
- JDK 11 или выше
- Maven

### Компиляция
```bash
mvn clean compile
```

### Запуск симуляции
Запускает кластер из 3-х нод, создает пользователей с балансами и симулирует нагрузку с хаос-тестированием (переключение лидера "на лету").
```bash
mvn exec:java -Dexec.mainClass="org.example.MainKt"
```

### Тесты
Запуск unit и интеграционных тестов (Matching, Replay, Concurrency):
```bash
mvn test
```

## 📊 Мониторинг
В процессе работы движок выводит метрики:
- **Throughput**: Команд в секунду.
- **Latency (p99)**: Время обработки одного ордера.
- **Total Matches**: Количество совершенных сделок.
